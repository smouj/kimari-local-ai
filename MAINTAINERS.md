## Responsibilities

The maintainer is responsible for:

- **Release management** — Version bumps, building, publishing to PyPI, tagging releases
- **Security response** — Triaging and fixing security vulnerabilities promptly
- **PR review and merge** — Reviewing all pull requests against acceptance criteria (see [GOVERNANCE.md](./GOVERNANCE.md))
- **Roadmap decisions** — Setting priorities and deciding what goes into each release
- **Technical quality and architecture** — Ensuring code quality, test coverage, and architectural consistency
- **Community health** — Enforcing the Code of Conduct, fostering a welcoming environment, responding to community issues

## How to Become a Maintainer

Kimari Local AI currently has a single maintainer. As the project grows, additional maintainers may be added based on:

1. **Consistent contributions** — A sustained history of high-quality PRs, reviews, and community involvement
2. **Trust** — Demonstrated reliability, good judgment, and alignment with project values
3. **Maintainer nomination** — Existing maintainers nominate candidates based on the above criteria

The process is intentionally lightweight during the alpha phase and will be formalized as the project and community grow.
